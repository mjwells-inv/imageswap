Video Placeholders Guide

This tool follows the same logic as the Images and Placeholders Guide, with one key difference. This tool is built on the assumption that videos are always intended by the model to be placeholders that the user will later update or replace. As far as we are aware, that assumption has been accurate for pretty much every task in this campaign so far. However, if you think that assumption might not be valid for a specific task, please post about it in the Slack channel and ask for guidance there.

What do you see where there should be a video?

Broken video: Any situation where the video does not successfully play should be included in this option. For example, there could be a video player frame but nothing happens or you get an error message when you click play, or there could be nothing visible at all.

Working video: If a video (any video) plays successfully, choose this option.


What is the video URL?

If the video works, you don't need to mess with this.
For broken videos, use "inspect" or "view page source" to find the URL or filename for the video. Be careful, because the video tag may also include a URL for the video's thumbnail image, but that's not the one you want.

URL is itself a placeholder: Choose this option if the webpage links to the video with a made-up URL or filename that is intended to be replaced by the user (like “video1.mp4” or “https://example.com/path/to/yourvideo.mp4”). If only part of the URL is intended to be replaced (like "https://www.youtube.com/embed/YourVideoID") that also counts.

Note:A local file name like "Path_to_video.mp4" will show up as "Path_to_video.mp4" when you use "inspect" or look in the source code. However, if you use "copy image address," it will look like "https://featherstorageprod-xxxxx.azurefd.net/assets/Path_to_video.mp4". So if you see a "featherstorage" URL, a placeholder file name was probably used.


Any Other Situation: Just like it says. If the URL/filename is anything other than a made-up placeholder clearly intended to be replaced, choose this.


Does the completion text say that placeholders were used?

We have been told that any time placeholders are used, the completion text must explain to the user that placeholders were used. The text does not need to use the word "placeholder." Any statement that the video URL or file name can or must be replaced by the user should count as a "yes," and Ctrl+f searching for the word "replace" is often the quickest way to find placeholder explanations.


Yes: We are not currently evaluating the quality or accuracy of the completion text, so choose "yes" if the completion text states that a placeholder was used for a video, even if part of the statement (such as the specific video used) isn't accurate.


No: Choose this if the text doesn't state that the user can or should replace the included video with their own. If the completion includes only a webpage link and no additional text, that always counts as a "no."


General guidelines for failure to acknowledge placeholders

Failure to acknowledge placeholders in the completion text should be considered a minor flaw. It may negatively affect functionality scores above 4, but it should never push a 4 down to a 3 or have any effect on scores below 4.
(However, it can always be used as an overall rating tie-breaker if there aren’t more significant differences to consider.)
